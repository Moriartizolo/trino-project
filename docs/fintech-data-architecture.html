Архитектура данных
fintech.transactions

Трёхслойное хранение транзакций: PostgreSQL (операционка), Iceberg/MinIO (data lake), ClickHouse (аналитика). Единая точка SQL — Trino.

Проект: trino-project · Июль 2026

Общая схема

Kafka (topic: transactions) ↓ kafka-consumer ├── PostgreSQL fintech.transactions ├── Iceberg/MinIO fintech.transaction_facts └── ClickHouse fintech.transaction_analytics Trino — SQL-доступ ко всем трём хранилищам
СлойХранилищеРоль
1PostgreSQLОперационный реестр — что произошло
2MinIO + IcebergData lake — полная копия всех полей
3ClickHouseАналитика — срез для Grafana
Связь: transaction_id — общий ключ во всех трёх таблицах.

INSERT → clickhouse.fintech.transaction_analytics (14 полей) 6. Trino — SQL-запросы и JOIN по transaction_id
